A simple DialogFlow fulfillment demonstration app that messes around with entities

What is it doing ?

The data structure that dialogflow sends to our fulfillment app looks like this: 

        
        { responseId: '5f9a9135-8409-459b-b79b-f1b8f977ad22',

          queryResult: 

           { queryText: 'I want to eat a banana with my long lost grand-mother',

             action: 'input.welcome',

             parameters: { fruits: 'banana' },

             allRequiredParamsPresent: true,

             fulfillmentText: 'Hi! How are you doing?',

             fulfillmentMessages: [ [Object] ],

             intent: 

              { name: 'projects/playing-around-with-entities/agent/intents/77f2fee9-f282-403b-a239-7e212506fecd',

                displayName: 'Default Welcome Intent' },

             intentDetectionConfidence: 0.85,

             languageCode: 'en' },

          originalDetectIntentRequest: { payload: {} },

          session: 'projects/playing-around-with-entities/agent/sessions/d5409e12-dc6d-8b72-ab6f-19eaab1ee16c' }

So we just need to play around with that data :)

Try typing something like I want to eat a banana with my dog. In the agent, I've only made a simple Fruits entity that is captured in the intent and sent to the fulfillment app.
